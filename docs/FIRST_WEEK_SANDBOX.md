# 第一週半沙盒規格（v0.1 Foundation）

> 後續開發請以 [第一週垂直切片鎖定企劃（v1.0）](./FIRST_WEEK_VERTICAL_SLICE_DESIGN.md) 為準；本文件保留作為早期方向與既有骨架紀錄。

## 目的

將目前「接客、休息、購物」的核心原型，轉成有明確壓力、自由日程與多種章末結果的半沙盒第一章。

本規格只定義第一週。玩家能自由安排每天的行動，但世界用期限、人物關係與事件節點維持方向。

## 玩家承諾

> 在七天內決定柯妤潔如何維持生活：高風險接客、較穩定的商店工作，或經營與阿坤及常客的關係。

第一週結束時，玩家應能清楚理解：
- 金錢、體力、名聲各自的用途。
- 接客、工作、休息、購物之間的取捨。
- 阿坤關係會改變可用選項與章末結果。
- 下一章會擴大城市與人物故事。

## 範圍

### 包含

- 第 1 至第 7 天的目標與結算。
- 開場說明、成人內容警示與新手教學。
- 主畫面的目標面板。
- 初期客人保護規則。
- 商店臨時工作。
- 阿坤的信任、好感、支配三項關係值。
- 第 3、5、7 天事件。
- 三種章末結果。

### 不包含

- 全部未開放地點。
- 完整懷孕、分娩、月經、玩具、野戰系統。
- 第二章以後的長期主線。
- 更多服裝、立繪或新背景。
- 線上帳號、排行、成就。

## 每日結構

保留現有分鐘制時間，但 UI 以四個時段協助理解：

| 時段 | 建議用途 |
| --- | --- |
| 早上 | 整理、工作、購物、拜訪人物 |
| 下午 | 工作、跑腿、外出事件 |
| 晚上 | 接客、人物事件 |
| 夜晚 | 休息或低風險私人事件 |

每個行動仍消耗實際分鐘與體力；時段只用於事件出現條件與玩家閱讀。

## 第一週目標

- 開始日：第 1 天。
- 截止日：第 7 天晚上。
- 暫定資金目標：1,200G。
- 目標數值必須在五次完整試玩後調整；不能以目前設定視為最終平衡。

目標面板至少顯示：

```
第一週：建立立足點
第 3 / 7 天
資金：680 / 1,200G
下一個提示：前往商店，詢問阿坤是否有工作。
```

## 新手保護

1. 前三位客人固定為 Tier 1。
2. 第一位客人明示偏好，並以文字提示「觀察 → 提升性奮 → 選擇服務」。
3. 第一日必須可在不冒險的情況下完成一次休息、一次購物或工作、一次存檔。
4. 任何初期失敗都不應造成永久卡關。

## 可用地點

| 地點 | 功能 | 代價／風險 |
| --- | --- | --- |
| 娼院 | 接客、休息、更衣 | 體力與時間 |
| 商店 | 購物、臨時工作、阿坤事件 | 時間、關係改變 |
| 家 | 睡眠、私人事件、章節整理 | 失去可賺錢時段 |
| 街區 | 跑腿、偶遇、常客事件 | 隨機性與時間 |

醫院、警局、巷弄、公廁保持不可用，直到各自有完整玩法閉環。

## 收入選擇

### 接客

- 潛在收入最高。
- 體力消耗與不確定性較高。
- 提升名聲、熟練度與部分常客關係。

### 商店臨時工作

- 2 小時。
- 固定收入 80 至 120G。
- 體力消耗低。
- 提升阿坤信任。
- 可能觸發加班、折扣、跑腿或特殊委託。

### 休息

- 不直接獲得收入。
- 恢復體力、降低壓力。
- 高壓力時應優先於接客。

## 新增狀態

第一週只新增兩組狀態：

```js
progress: {
  chapter: 'first_week',
  deadlineDay: 7,
  targetGold: 1200,
  tutorialStep: 0,
  completedMilestones: [],
  resolved: false
},

relationships: {
  boss: {
    affection: 0,
    trust: 0,
    dominance: 0,
  },
}
```

不要在第一週加入大量新屬性。壓力值可以在第二個實作階段加入；第一階段先用既有體力與關係值驗證循環。

## 事件節點

### 第 1 天：開場與教學

- 說明期限與資金目標。
- 引導完成第一位客人或第一份商店工作。
- 引導休息、購物或裝備、存檔。

### 第 3 天：阿坤試探

觸發條件：前往商店，或資金低於目標的 35%。

選擇：
- 接受商店工作：收入穩定，信任上升。
- 直接要求折扣：有機率影響好感與支配。
- 拒絕：保留時間，但失去一次穩定收入。

### 第 5 天：常客或壓力事件

觸發條件：名聲、接客次數或阿坤關係達到門檻。

目的：讓玩家看見「重複選擇會讓人物記得你」，而非只有隨機客人。

### 第 7 天：結算

在截止日晚上檢查金錢與阿坤關係。

| 結果 | 條件 | 下一章效果 |
| --- | --- | --- |
| 自立起步 | 金錢達標 | 解鎖更高收入與常客線 |
| 有條件延期 | 金錢不足，但阿坤信任足夠 | 債務減輕，阿坤支配提高 |
| 債務加重 | 金錢不足且關係不足 | 解鎖受限制但可逆轉的第二章路線 |

不做硬性 Game Over。失敗必須產生可玩的後果。

## 實作順序

1. 加入 `src/progression.js` 的純邏輯。
2. 為該邏輯新增測試。
3. 擴充存檔遷移，補齊 `progress` 與 `relationships`。
4. 實作目標面板。
5. 實作開場與四步教學。
6. 實作 Tier 1 新手客人保護。
7. 實作商店臨時工作。
8. 實作第 3、5、7 天事件與結算。
9. 以五次完整試玩調整資金目標與報酬。

## 驗收標準

- 新玩家在一分鐘內說得出第一週目標。
- 玩家在第一天知道如何賺錢、恢復體力、前往商店與存檔。
- 第一週至少有三種不同結果。
- 沒有任何收入、體力或時間組合造成永久卡關。
- 完成第一週的中位遊玩時間介於 30 至 60 分鐘。
